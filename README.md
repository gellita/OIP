# Задание 4

# TF-IDF для терминов и лемм (папка task4)

Программа вычисляет **TF-IDF** для каждого скачанного документа из задания 1.

Считается:

- `tf` для каждого **термина (токена)** документа
- `idf` для каждого термина по всей коллекции
- `tf` для каждой **леммы** как отношение суммы вхождений терминов этой леммы к общему количеству терминов в документе
- `idf` для каждой леммы по всей коллекции

Результаты сохраняются **по каждому документу** в формате:

`<термин> <idf> <tf-idf>\n`  
`<лемма> <idf> <tf-idf>\n`

---

## Требования

Для запуска необходимы:

- **Python 3.10** или выше
- установленный **pip**

---

## Установка зависимостей

Перед запуском установите необходимые библиотеки:

```bash
pip install pymorphy3
```

---

## Запуск программы

1. Откройте терминал.
2. Перейдите в папку проекта `task4`:
   ```bash
   cd путь_к_папке_проекта/task4
   ```
3. Запустите скрипт:
   ```bash
   python tfidf.py
   ```

После запуска программа:

- посчитает `df` и `idf` по всей коллекции документов
- для каждого документа сформирует файлы с TF-IDF

---

## Результаты

В папке `task4` создаются две директории:

- `tfidf_terms/` — TF-IDF по терминам (токенам) для каждого документа
- `tfidf_lemmas/` — TF-IDF по леммам для каждого документа

Файлы соответствуют документам:

- `tfidf_terms/terms1.txt`, `tfidf_lemmas/lemmas1.txt`
- `tfidf_terms/terms2.txt`, `tfidf_lemmas/lemmas2.txt`
- и т.д.

---

## Формат выходных файлов

### `tfidf_terms/termsN.txt`

Каждая строка:

`<термин> <idf> <tf-idf>`

Пример:

```txt
алебастра 4.60517019 0.00865633
авторы 0.00000000 0.00000000
```

### `tfidf_lemmas/lemmasN.txt`

Каждая строка:

`<лемма> <idf> <tf-idf>`

Пример:

```txt
алебастр 4.60517019 0.00865633
автор 0.00000000 0.00000000
```

---

## Примечания

Используются формулы:

- `tf(term, doc) = count(term, doc) / total_terms(doc)`
- `idf(term) = ln(N / df(term))`
- `tf-idf = tf * idf`

# Задание 5

# Векторный поиск

В данном задании реализована поисковая система на основе **векторного поиска**.  
Документы и запрос представляются в виде **TF-IDF векторов по леммам**, а релевантность вычисляется через **косинусное сходство** (cosine similarity).

Источник данных для векторов — результаты **Задания 4**:

- `task4/tfidf_lemmas/lemmasN.txt` (формат строк: `<лемма> <idf> <tf-idf>`)

---

## Требования

Для запуска необходимы:

- **Python 3.10** или выше
- установленный **pip**

---

## Зависимости

### Для CLI

```bash
pip install pymorphy3
```

---

## Структура проекта

Ожидаемая структура (минимально необходимая для работы задания 5):

```txt
OIP/
  task1/
    index.txt                 (doc_id -> URL)
  task4/
    tfidf_lemmas/
      lemmas1.txt
      lemmas2.txt
      ...
  task5/
    engine.py                 (ядро поиска)
    cli.py                    (консольное приложение)
```

---

## Как работает поиск

1. Загружаются TF-IDF файлы документов из `task4/tfidf_lemmas/`.
2. Для запроса строится TF-IDF вектор по леммам (IDF берётся из корпуса).
3. Считается косинусное сходство запроса с документами.
4. Возвращается **Top-K** документов по убыванию score.

---

## Запуск CLI

1. Есть файлы `task4/tfidf_lemmas/lemmasN.txt`.
2. Открыть терминал и перейти в папку `task5`:

```bash
cd путь_к_папке_проекта/task5
```

3. Запустить консольное приложение:

```bash
python cli.py
```

4. Введите запрос:

```txt
QUERY> клеопатра цезарь
```

Выход — пустая строка.

---

### Режим Strict

В интерфейсе есть переключатель **Strict**:

- `Strict = OFF` — выдаёт и частичные совпадения (похожие документы по пересечению лемм)
- `Strict = ON` — документ должен содержать **все леммы** запроса (AND-фильтр), затем ранжирование
